# AlgoStudio Web Console 用户手册

## 概述

AlgoStudio Web Console 是 AI 算法平台的可视化管理界面，提供任务管理、主机监控和算法部署功能。

**访问地址**: `http://localhost:3000`
**API 地址**: `http://localhost:8000` (后端服务)

---

## 导航说明

顶部导航栏提供以下页面跳转:

| 页面 | 路径 | 功能 |
|------|------|------|
| Dashboard | `/` | 平台概览、集群状态、资源图表 |
| 任务列表 | `/tasks` | 训练/推理/验证任务管理 |
| 主机监控 | `/hosts` | Ray 集群节点状态监控 |
| 部署算法 | `/deploy` | 算法部署到集群节点 |

---

## 1. Dashboard (仪表盘)

访问地址: `/`

Dashboard 提供平台全局概览，包括:

### 1.1 统计卡片

- **任务总数**: 当前所有任务数量
- **运行中**: 正在执行的任务数
- **待处理**: 等待调度的任务数
- **失败**: 执行失败的任务数

### 1.2 集群状态

显示 Ray 集群的整体状态，包括在线/离线节点数量。

### 1.3 资源图表

展示集群整体 CPU、GPU、内存使用情况的可视化图表。

### 1.4 最近任务

显示最近创建的 5 个任务，点击可跳转至任务详情。

---

## 2. 任务管理

### 2.1 任务列表

访问地址: `/tasks`

**功能说明**:
- 查看所有训练、推理、验证任务
- 按状态筛选任务 (待处理/运行中/已完成/失败/已取消)
- 搜索任务 ID 或算法名称
- 分页浏览任务列表

**任务状态说明**:

| 状态 | 中文 | 说明 |
|------|------|------|
| pending | 待处理 | 任务已创建，等待调度 |
| running | 运行中 | 任务正在执行 |
| completed | 已完成 | 任务成功完成 |
| failed | 失败 | 任务执行失败 |
| cancelled | 已取消 | 任务被手动取消 |

**任务类型说明**:

| 类型 | 中文 | 说明 |
|------|------|------|
| train | 训练 | 模型训练任务 |
| infer | 推理 | 模型推理任务 |
| verify | 验证 | 模型验证任务 |

**列表字段说明**:

| 字段 | 说明 |
|------|------|
| 任务ID | 任务的唯一标识符 |
| 类型 | 任务类型 (训练/推理/验证) |
| 算法 | 算法名称和版本 |
| 状态 | 当前任务状态 |
| 进度 | 任务完成百分比 |
| 节点 | 分配的执行节点 IP |
| 创建时间 | 任务创建的时间 |

### 2.2 任务详情

访问地址: `/tasks/[taskId]`

**功能说明**:
- 查看任务的完整信息
- 实时进度展示 (运行中的任务)
- 查看错误信息 (失败的任务)

**页面信息**:

| 区块 | 内容 |
|------|------|
| 基本信息 | 任务类型、算法名称/版本、创建/开始/完成时间 |
| 执行信息 | 分配的节点、错误信息 |
| 进度 | 当前进度条和百分比 (仅运行中任务显示) |

**SSE 实时更新**:
任务详情页面通过 Server-Sent Events (SSE) 实时接收任务进度更新，无需刷新页面。

## 3. 主机监控

### 3.1 主机列表

访问地址: `/hosts`

**功能说明**:
- 查看 Ray 集群所有节点
- 显示节点在线/离线状态
- 按 IP 地址或主机名搜索
- 刷新节点状态

**状态说明**:

| 状态 | 说明 |
|------|------|
| 在线 (绿色) | 节点正常运行，已连接集群 |
| 离线 (红色) | 节点不可达或未连接集群 |

**主机卡片信息**:

| 字段 | 说明 |
|------|------|
| IP 地址 | 节点网络地址 |
| 主机名 | 节点主机名称 |
| CPU | CPU 型号、核心数、使用率 |
| GPU | GPU 型号、显存使用情况 |
| 内存 | 内存使用情况 |
| 状态 | 在线/离线标识 |

### 3.2 节点详情

访问地址: `/hosts/[nodeId]`

点击主机卡片可进入节点详情页面，查看完整资源信息:

| 资源 | 显示信息 |
|------|----------|
| CPU | 型号、核心数、使用率、频率 |
| GPU | 型号、数量、使用率、显存 |
| 内存 | 已用/总量 |
| 磁盘 | 已用/总量 (如有) |

## 4. 部署管理

访问地址: `/deploy`

### 4.1 部署向导

部署算法到集群节点采用三步向导流程:

**步骤 1: 选择算法**
- 选择要部署的算法名称
- 选择算法版本 (v1, v2 等)

**步骤 2: 选择主机**
- 从在线节点列表中选择目标主机
- 显示选中主机的详细信息 (IP、GPU 型号、显存)

**步骤 3: 配置部署**
- 启动 Ray Worker: 是否在部署后启动 Ray Worker 进程
- 故障时自动重启: 进程异常退出时自动重启
- GPU 内存限制: 设置 GPU 显存使用上限

### 4.2 部署进度

点击"开始部署"后，弹出部署进度对话框:

**显示内容**:
- 当前部署步骤
- 整体进度百分比
- 部署日志 (实时更新)

**部署步骤说明**:

| 步骤 | 说明 |
|------|------|
| 连接主机 | 建立与目标节点的 SSH 连接 |
| 同步算法文件 | 将算法代码复制到远程主机 |
| 安装依赖 | 在远程环境安装算法依赖 |
| 配置 GPU | 设置 GPU 相关参数 |
| 启动 Ray Worker | 启动 Ray Worker 进程 |
| 验证部署 | 确认部署成功 |

### 4.3 部署历史

部署任务会创建对应的任务记录，可在 `/tasks` 页面查看部署任务的执行状态。

## 5. 常见操作流程

### 5.1 查看任务执行进度

1. 进入 `/tasks` 页面
2. 在任务列表中找到目标任务
3. 点击任务 ID 进入详情页面
4. 页面会通过 SSE 自动更新进度

### 5.2 监控集群节点状态

1. 进入 `/hosts` 页面
2. 查看所有节点的状态概览
3. 点击卡片查看节点详细信息 (CPU/GPU/内存)
4. 使用刷新按钮更新状态

### 5.3 部署算法到新节点

1. 进入 `/deploy` 页面
2. **步骤 1**: 选择算法和版本
3. **步骤 2**: 从在线节点中选择目标主机
4. **步骤 3**: 配置部署选项 (Ray Worker 启动等)
5. 点击"开始部署"
6. 在弹出的进度对话框中查看部署状态
7. 部署完成后可前往 `/hosts` 确认节点状态

### 5.4 排查任务失败原因

1. 进入 `/tasks` 页面
2. 找到状态为"失败"的任务
3. 点击任务 ID 进入详情
4. 查看"错误信息"区域的具体错误描述
5. 根据错误信息进行相应修复

### 5.5 搜索特定任务

1. 进入 `/tasks` 页面
2. 在搜索框中输入任务 ID 或算法名称
3. 结果会实时过滤显示

### 5.6 按状态筛选任务

1. 进入 `/tasks` 页面
2. 点击"任务状态"下拉框
3. 选择目标状态 (如"运行中")
4. 列表会自动筛选显示匹配的任务

## 6. 页面刷新说明

所有列表页面支持手动刷新:

- **自动刷新**: 页面每 30 秒自动刷新数据
- **手动刷新**: 点击列表上方的刷新图标按钮
- **强制刷新**: 浏览器 F5 键或 Ctrl+R

## 8. 注意事项

1. **Head 节点标识**: 在主机列表中，Head 节点会显示 "Head 节点" 标签
2. **离线节点**: 离线节点无法接收新任务调度
3. **SSE 连接**: 任务详情页面的实时更新依赖 SSE 连接，确保网络畅通
4. **部署权限**: 部署功能需要有效的 SSH 凭证配置
5. **浏览器兼容性**: 推荐使用 Chrome、Firefox、Edge 最新版本
