# Phase 2 待办列表

> AI 算法平台 Phase 2 功能待开发事项

---

## 高优先级

### 1. SSH 自动部署 Worker 节点
**描述：** 在 Web Console 中输入目标主机 IP、SSH 密码，通过 SSH 自动完成 Worker 节点部署

**功能点：**
- Web 界面：输入 Worker 节点 IP、SSH 用户名/密码、代码路径
- 后端 SSH 连接：使用 `paramiko` 库实现安全的 SSH 连接
- 远程脚本执行：自动 clone 代码、执行 `join_cluster.sh`
- 部署进度显示：实时显示部署状态（连接中、安装中、启动中、完成）
- 异常处理：连接失败、认证失败、安装失败的错误提示

**技术依赖：**
- `paramiko` SSH 库
- Web 后端异步任务处理（显示进度）
- 部署状态 API

**相关文档：**
- `docs/cluster-deployment.md` - 现有部署流程

---

### 2. 任务历史与日志查看
**描述：** 查看历史任务执行记录和详细日志

**功能点：**
- 任务列表：分页展示历史任务，包含状态、时间、算法名称
- 任务详情：查看任务配置、输出日志、执行时长
- 日志流：实时查看任务日志（类似 `tail -f`）
- 任务重试：失败任务一键重试

---

### 3. 算法版本管理
**描述：** 支持同一算法的多个版本管理和切换

**功能点：**
- 版本列表：显示算法所有版本（v1、v2...）
- 版本对比：对比不同版本的配置和性能
- 默认版本设置：设置默认使用的版本
- 版本回滚：切换回旧版本

---

## 中优先级

### 4. 任务告警与通知
**描述：** 任务完成或失败时发送通知

**功能点：**
- 告警规则：任务失败、GPU 内存不足、节点离线等
- 通知渠道：Web UI 提醒、邮件通知（可选）
- 告警历史：查看历史告警记录

---

### 5. 资源配额管理
**描述：** 为不同用户/团队设置资源使用配额

**功能点：**
- 配额设置：CPU、GPU、内存使用上限
- 配额监控：接近配额时告警
- 配额占用视图：查看各用户资源使用情况

---

### 6. 数据集管理增强
**描述：** 增强 Phase 1 的数据集管理功能

**功能点：**
- 数据集预览：图片、文本、视频等类型预览
- 数据集版本：基于 DVC 的版本管理
- 数据集分享：跨项目数据集共享

---

## 低优先级

### 7. 用户权限管理
**描述：** 多用户支持和权限控制

**功能点：**
- 用户注册/登录（可选，初期可跳过）
- 角色：管理员、普通用户
- 权限：只有管理员能添加/删除 Worker

---

### 8. 任务结果可视化
**描述：** 训练/推理结果图表展示

**功能点：**
- 训练曲线：loss、accuracy 等指标曲线
- 推理结果：图片检测结果可视化
- 性能对比：多算法性能对比图表

---

### 9. API 文档自动化
**描述：** 自动生成和展示 API 文档

**功能点：**
- Swagger UI：FastAPI 自动生成的 API 文档
- API 密钥管理：第三方应用调用接口

## 备注

- Phase 2 优先级由用户需求驱动，高优先级功能应尽快实现
- SSH 自动部署是简化运维的关键功能，应在早期完成
