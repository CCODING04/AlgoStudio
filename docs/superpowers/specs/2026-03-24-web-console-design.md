# Web Console 设计文档

**项目名称：** AlgoStudio Web Console
**日期：** 2026-03-24
**状态：** 设计中
**范围：** Phase 2 Plan A（3 页面 Gradio 应用）

---

## 1. 概述

### 1.1 目标

使用 Gradio 构建 Web Console，为 AlgoStudio 提供可视化界面，用于：
1. 检查 Phase 1 实现（任务、主机监控）
2. 作为后续 Web 功能扩展的基础

### 1.2 技术栈

- **UI 框架：** Gradio 5.x（Python）
- **后端 API：** FastAPI（复用 Phase 1）
- **架构：** 前后端分离，API 层（FastAPI）和 UI 层（Gradio）解耦

### 1.3 目录结构

```
algo_studio/web/
├── __init__.py
├── app.py              # Gradio APP 入口，Blocks 组装
├── config.py           # 主题、布局配置
└── pages/
    ├── __init__.py
    ├── dashboard.py    # 仪表盘（首页）
    ├── tasks.py        # 任务列表
    └── hosts.py        # 主机监控
```

## 3. 响应式设计

### 3.1 布局策略

- **Gradio 配置：** `fluid=True`，CSS `max-width: 100%`
- **容器宽度：** `width: 100%` 弹性布局
- **Gradio Blocks：** 使用 `with gr.Column()` 实现响应式列

### 3.2 断点

| 设备 | 宽度 | 机器卡片列数 |
|------|------|-------------|
| 手机 | < 640px | 1 列 |
| 平板 | 640-1024px | 2 列 |
| 桌面 | > 1024px | 3-4 列 |

### 3.3 Gradio 实现

```python
with gr.Column(scale=1, min_width=300):
    # 卡片内容
```

## 4. 技术实现

### 4.1 Gradio Blocks vs Blocks Theme

使用 **Gradio Blocks**（而非 `gradio.Blocks()`）：
- 更灵活的布局控制
- 支持自定义 CSS
- 方便实现响应式

### 4.2 API 客户端

复用 `algo_studio.api.routes.tasks` 和 `hosts` 的 requests 调用：

```python
import requests
API_BASE = os.environ.get("ALGO_STUDIO_API", "http://localhost:8000")

def get_tasks():
    resp = requests.get(f"{API_BASE}/api/tasks", timeout=10)
    return resp.json()

def get_hosts_status():
    resp = requests.get(f"{API_BASE}/api/hosts/status", timeout=10)
    return resp.json()
```

### 4.3 主题

使用 Gradio 内置 `theme` 或自定义 CSS：
- 简洁专业风格
- 配合 AlgoStudio 品牌色

### 4.4 自动刷新

- Gradio `Timer()` 组件
- 间隔 30 秒触发数据刷新
- Toggle 控制开关
