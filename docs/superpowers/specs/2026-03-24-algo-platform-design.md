## 1. 整体架构

### 1.1 架构分层

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户交互层                                │
│                   Web 控制台  +  CLI 工具                        │
└─────────────────────────────────────────────────────────────────┘
                                │
┌─────────────────────────────────────────────────────────────────┐
│                       API 网关层                                 │
│                    FastAPI + 任务队列                            │
└─────────────────────────────────────────────────────────────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│ AI 集成核心   │ ──→ │  算法平台核心  │     │  主机管理     │
│  (Dev)       │     │  (Stable)    │     │  Ray Cluster  │
│               │     │  训练/推理   │     │              │

                    ┌───────────────────────┐
                    │      数据集层          │
                    │   NAS + DVC 版本管理   │
                    └───────────────────────┘
```

### 1.2 子系统定位

| 子系统 | 定位 | 职责 |
|--------|------|------|
| AI 集成核心 (Dev) | 开发/进化 | 算法的自我迭代进化、备份管理、演进日志 |
| 算法平台核心 (Stable) | 生产 | 稳定运行的算法库，训练/推理/验证 |
| 主机管理 | 基础设施 | Ray Cluster 多机调度、状态监控 |
| 数据集层 | 基础设施 | NAS 存储 + DVC 版本管理 |

### 1.3 核心流程

```
AI 集成核心（Dev）进化算法 → 评审通过 → 推送到算法平台（Stable）稳定运行
```

## 2. AI 集成核心（Dev）

### 2.1 Multi-Agent 架构

```
                    ┌─────────────────┐
                    │  Supervisor     │
                    │  Agent          │
                    │  (协调者)       │
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          ▼                  ▼                  ▼
   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
   │ Research    │    │  Develop    │    │ Validation  │
   │ Agent       │    │  Agent      │    │ Agent       │
   │ (深度研究)   │    │  (代码开发)  │    │ (验证评测)  │
   └─────────────┘    └─────────────┘    └─────────────┘
```

**各 Agent 职责：**
- **Research Agent** — 分析需求，探索方案，做深度研究
- **Develop Agent** — 生成/修改代码，实现算法
- **Validation Agent** — 评测结果，决定是否达标或需要迭代

### 2.2 演进日志（Evolution Log）

演进日志增量追加，每次迭代增加新记录：

```json
{
  "iteration_id": "algo_vision_001",
  "timestamp": "2026-03-20T10:00:00Z",
  "agent": "research",
  "action": "analyzed_requirements",
  "input": { "scenario": "边端部署", "fps": 30, "task": "目标检测" },
  "output": { "baseline_approach": "YOLO-family", "estimated_fps": 45 },
  "gaps": ["精度待提升", "模型过大"],
  "next_steps": ["尝试知识蒸馏", "剪枝优化"]
}
```

### 2.3 实验记录（Experiment Record）

每次实验独立记录：

```yaml
# experiments/run_001/config.yaml
experiment_id: run_001
purpose: 验证 YOLO-family 边端部署可行性
environment:
  algo_version: v1.0.0
  dataset: coco_2024
  hardware: RTX 4090
parameters:
  epochs: 100
  batch_size: 16
results:
  mAP: 0.78
  FPS: 35
conclusion: 初步满足需求，建议继续优化精度
next_steps:
  - 尝试知识蒸馏提升精度
  - 剪枝优化降低延迟
```

### 2.4 Git 分支模型

```
┌─────────────┐         ┌─────────────┐         ┌───────────┐
│    dev      │  merge  │   staging   │  merge  │  stable   │
│   (AI 工作)  │ ──────→ │  (审核区)    │ ──────→ │  (生产)   │
└─────────────┘         └─────────────┘         └───────────┘
      │                       │                         │
      ▼                       ▼                         ▼
演进日志追加              人/Agent 审核             算法平台同步
实验记录写入              合并确认
```

### 2.5 人 + AI 协作模式

**两种协作入口：**
1. AI 主导：AI 迭代 → push → 人 review → Agent 验证 → merge
2. 人主导：人 push 修改/备注 → Agent 检查合理性 → 人确认 merge

**Agent 审核内容：**
- 代码规范检查
- 训练结果合理性验证
- 与已有知识的冲突检测
- 配置文件完整性

## 3. 算法平台核心（Stable）

### 3.1 模块结构

```
┌─────────────────────────────────────────────────────────────────┐
│                    算法平台核心 (Stable)                         │

│  │               │  │               │  │               │       │
│  │ - 接口规范    │  │ - 训练任务    │  │ - 批量推理    │       │
│  │ - 版本管理    │  │ - 资源调度    │  │ - 在线服务    │       │
│  │ - 元数据      │  │ - 进度追踪    │  │ - 批处理      │       │
│  └───────────────┘  └───────────────┘  └───────────────┘       │
│  ┌───────────────┐  ┌───────────────────────────────────┐       │
│  │   验证模块    │  │         算法仓库                   │       │
│  │               │  │                                   │       │
│  │ - 接口校验    │  │  /algorithms/                     │       │
│  │ - 性能基准    │  │    ├── yolo_family/                │       │
│  │ - 准入测试    │  │    │   ├── v1.0/                    │       │
│  │               │  │    │   └── v2.0/                    │       │
│  └───────────────┘  │    │       ├── model.onnx          │       │
│                     │    │       ├── config.yaml         │       │

│                     │    └── other_algorithm/            │       │
│                     └───────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 算法接口规范

```python
class AlgorithmInterface:
    def train(data_path: str, config: dict) -> TrainResult:
        """训练接口"""

    def infer(inputs: list) -> list:
        """推理接口"""

    def verify(test_data: str) -> VerificationResult:
        """验证接口"""

    def get_metadata() -> AlgorithmMetadata:
        """返回算法元信息"""
```

### 3.3 算法仓库结构

```
/algorithms/<algo_name>/
├── v<version>/
│   ├── src/                    # 核心实现
│   │   ├── __init__.py
│   │   ├── model.py           # 模型定义
│   │   ├── backbone.py        # backbone 网络
│   │   ├── head.py            # 检测头等
│   │   └── loss.py            # 损失函数
│   │
│   ├── train.py               # 训练入口（调用 src/）
│   ├── infer.py               # 推理入口（调用 src/）
│   ├── validate.py            # 验证入口
│   │
│   ├── config.yaml            # 配置文件
│   ├── metadata.json          # 元信息
│   └── requirements.txt       # 依赖
│
└── weights/                    # 权重独立存储
    └── run_001_train_v1.0.1_epoch50.pth
```

### 3.4 权重元信息

```json
{
  "weight_file": "run_001_train_v1.0.1_epoch50.pth",
  "trained_with_algo_version": "1.0.1",
  "trained_on_dataset": "/nas/datasets/coco_2024",
  "epochs": 50,
  "metrics": { "mAP": 0.78, "FPS": 35 },
  "compatible_with": ["1.0.1"],
  "incompatible_reason": null
}
```

**兼容性规则：**
- `compatible_with` 列明该权重兼容哪些算法版本
- 如果当前运行的算法版本不在列表中，平台提示不兼容
- 用户可选择：回滚算法 或 迁移训练

## 4. 主机管理（Ray Cluster）

### 4.1 架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      主机管理 (Ray Cluster)                      │
├─────────────────────────────────────────────────────────────────┤
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                    Ray Head Node                        │   │
│   │                   (调度协调者)                           │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│          ┌───────────────────┼───────────────────┐             │
│          ▼                   ▼                   ▼             │
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐      │
│   │  Worker 1   │     │  Worker 2   │     │  Worker N   │      │
│   │  (开发/训练) │     │  (推理)     │     │  (空闲调用)  │      │
│   └─────────────┘     └─────────────┘     └─────────────┘      │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 调度策略

- **空闲优先** — 哪个机器闲用哪个
- **资源匹配** — 训练用 GPU，推理可用 CPU
- **可指定机器** — 必要时可指定特定机器运行

## 7. 实现方案

### 7.1 核心技术栈

| 层级 | 技术选型 |
|------|---------|
| API/任务队列 | FastAPI + Redis/RQ |
| AI Agent | Claude API + Python 实现 |
| 任务调度 | Ray |
| 算法仓库 | Git + 文件系统 |
| 数据集 | DVC + NAS |
| Web UI | Gradio 或 React（待定） |
| CLI | Python Click 或 Typer |

### 7.2 实现阶段

**Phase 1（优先）：**
1. Ray Cluster 部署（3-5 台机器互通）
2. 基础算法接口定义 + 算法仓库
3. 训练/推理任务提交和调度
4. 基础主机状态监控

**Phase 2：**
5. AI Agent 集成
6. 演进日志 + Git 分支模型
7. Web 控制台
